Paint.NET Deployment via Group Policy
-------------------------------------
If your deployment requires or is simplified by having an MSI file, you may 
extract the Paint.NET installation MSI and get it to install by following 
these instructions. It is assumed that you are familiar with deployment via 
Group Policy, and the installation of MSI files.

1. Get the PaintDotNet.msi file.

   Run PaintDotNet_2_2.exe. The setup wizard will start. At this point, go to
   Start, then Run, and type "%TEMP%\PdnSetup" and then click OK. The MSI is
   in this directory until you close the setup wizard. Simply copy the file to
   wherever you want, then close the setup wizard by clicking the Cancel
   button.
      
2. Determine the options you want to use for installation.

   The MSI requires several properties to be set:
   
   TARGETDIR -- The target installation directory.
   CHECKFORUPDATES -- 0 or 1. Updates will only be checked for if Paint.NET is
       run by an Administrator.
   CHECKFORBETAS -- 0 or 1. Also checks for beta releases.
   JPGPNGBMPEDITOR -- 0 or 1. Defines whether or not Paint.NET is installed as
       the default editor for JPEG, PNG, and BMP images.
   TGAEDITOR -- 0 or 1. Defines whether or not Paint.NET is installed as the
       default editor for TGA images.
   FRONTEND -- 1. If this property is not 1, the MSI will refuse to execute.
   
   These are generally set via command-line parameters in the form of
   PROPERTY=VALUE. For example:
   
   "TARGETDIR=C:\Program Files\Paint.NET" CHECKFORUPDATES=1 CHECKFORBETAS=1
       JPGPNGBMPEDITOR=1 TGAEDITOR=1 FRONTEND=1
   
3. Deploy with "Always install with elevated privileges".
